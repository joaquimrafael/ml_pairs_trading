CONTEXTO GERAL
──────────────
O sistema de pairs trading implementado opera comprando um ativo e vendendo outro
simultaneamente, apostando que a diferença de preços entre eles (chamada de spread)
vai retornar à sua média histórica. Para que essa estratégia seja válida, é necessário
que os dois ativos tenham uma relação estatística estável no longo prazo.

Antes das alterações descritas neste documento, o sistema escolhia os pares por
conhecimento de domínio (PETR3/PETR4 são da mesma empresa, logo devem andar juntos)
e utilizava parâmetros fixos e arbitrários para configurar os modelos e as regras de
negociação. As alterações implementadas substituem essas escolhas arbitrárias por
valores derivados de evidência estatística extraída diretamente dos dados de cada par.

Todas as alterações são executadas automaticamente antes do treinamento dos modelos,
sem necessidade de intervenção manual. O pipeline completo é: análise estatística do
par → configuração dos parâmetros → treinamento do modelo → simulação das estratégias.


================================================================================
ALTERAÇÃO 1 — VALIDAÇÃO ESTATÍSTICA DO PAR (PRÉ-EXISTENTE, EXPANDIDA)
================================================================================

PROBLEMA QUE RESOLVE
─────────────────────
A escolha dos pares era justificada apenas por conhecimento de domínio. O professor
avaliador sinalizou a necessidade de justificativa estatística formal para a seleção
dos pares. Sem isso, não há como diferenciar um par que genuinamente reverte à média
de um par que apenas parece fazê-lo por coincidência no período analisado.

TRÊS TESTES APLICADOS
──────────────────────
1. Correlação de Pearson
   Mede se os dois preços se movem juntos no curto prazo. Uma correlação alta (acima
   de 0,90) indica que os ativos respondem aos mesmos fatores de mercado. Para
   PETR3/PETR4 obteve-se r = 0,9801, o que é esperado por serem ações ordinárias e
   preferenciais da mesma empresa.

2. Teste de Engle-Granger
   Verifica se os dois preços são cointegrados, ou seja, se existe uma combinação
   linear entre eles que é estacionária no longo prazo, mesmo que cada série
   individualmente seja não-estacionária. Na prática, pergunta: quando os preços se
   afastam, eles tendem a voltar para uma proporção estável? Para PETR3/PETR4 o
   p-valor foi 0,014, abaixo do limiar padrão de 0,05, confirmando cointegração.
   O teste não foi significativo ao nível de 1% (p < 0,01), o que é normal em dados
   intraday de alta frequência, que têm mais ruído que dados diários.

3. Teste ADF no Spread
   Aplica o teste de Dickey-Fuller Aumentado diretamente na série do spread (diferença
   entre os preços). Um spread estacionário significa que ele oscila em torno de uma
   média fixa em vez de derivar indefinidamente. Para PETR3/PETR4 o p-valor foi
   0,003, significativo ao nível de 1%, com 28 defasagens selecionadas automaticamente
   pelo critério AIC. Esse é o resultado mais diretamente relevante para o trabalho:
   ele justifica matematicamente todas as estratégias de reversão à média utilizadas
   no simulador.

RELAÇÃO COM O TRABALHO
───────────────────────
Os três testes formam uma cadeia de evidências: os ativos se movem juntos no curto
prazo (Pearson), têm relação de equilíbrio no longo prazo (Engle-Granger) e a
diferença entre eles é estacionária (ADF). Juntos, esses resultados transformam a
justificativa do par de "intuição de mercado" em "evidência estatística formal",
respondendo diretamente ao feedback do professor avaliador.

ALTERAÇÃO 2 — HEDGE RATIO β NO CÁLCULO DO SPREAD
================================================================================

PROBLEMA QUE RESOLVE
─────────────────────
O spread entre dois ativos não é simplesmente a diferença aritmética de seus preços.
Quando se constrói uma posição de pairs trading, vende-se um ativo e compra-se o
outro em quantidades que se cancelam mutuamente, tornando a posição neutra ao
movimento geral do mercado. A quantidade certa de cada ativo para atingir essa
neutralidade é chamada de hedge ratio.

O sistema original calculava o spread como "preço_A menos preço_B", assumindo
implicitamente que uma unidade monetária de A neutraliza exatamente uma unidade
monetária de B (hedge ratio igual a 1). Essa suposição é raramente correta na prática.

COMO FUNCIONA
─────────────
O teste de Engle-Granger, além de verificar a cointegração, estima por regressão
de mínimos quadrados ordinários (OLS) o coeficiente que minimiza a variância dos
resíduos da relação entre os dois preços. Esse coeficiente é o hedge ratio β.

A regressão estima: preço_A = α + β × preço_B + erro

O valor de β encontrado para PETR3/PETR4 foi 0,9924. Isso significa que, para cada
unidade de PETR3, a posição neutra exige 0,9924 unidades de PETR4, e não exatamente
1 unidade como o sistema assumia antes.

O spread corrigido passa a ser calculado como:
spread = preço_A − β × preço_B

Em vez de:
spread = preço_A − preço_B (equivalente a β = 1)

JUSTIFICATIVA ESTATÍSTICA
─────────────────────────
Usar β = 1 quando o valor real é 0,9924 introduz um viés sistemático no spread. Esse
viés é pequeno para PETR3/PETR4 porque ambas são da mesma empresa e têm preços
próximos. Para outros pares, a diferença pode ser maior. Mais importante: o spread
calculado com β = 1 carrega uma componente que não reverte à média, comprometendo a
premissa estatística de estacionariedade que justifica toda a estratégia.

EFEITO NO SISTEMA
─────────────────
O spread ajustado por β é utilizado em todas as partes do sistema: no cálculo do
z-score normalizado que alimenta o modelo de aprendizado por reforço, nas estratégias
de reversão à média do simulador, e nos gráficos de análise do par. O hedge ratio β é
reportado no relatório de análise impresso no console para cada execução.

Para PETR3/PETR4, β ≈ 0,99 é uma correção sutil. Para pares de setores distintos ou
com grandes diferenças de preço nominal, β pode diferir significativamente de 1 e a
correção torna-se essencial para a validade estatística da estratégia.


================================================================================
ALTERAÇÃO 3 — HALF-LIFE DA REVERSÃO À MÉDIA COMO JANELA DE ENTRADA
================================================================================

PROBLEMA QUE RESOLVE
─────────────────────
O parâmetro INPUT_CHUNK_LENGTH define quantas barras de histórico o modelo utiliza
para fazer cada previsão, e também o tamanho da janela deslizante usada para
normalizar o spread. O valor padrão era 50 barras, escolhido sem justificativa
estatística. Diferentes pares têm velocidades de reversão à média distintas, e usar
a mesma janela para todos introduz um desalinhamento entre a escala temporal do
fenômeno e a escala do modelo.

COMO FUNCIONA
─────────────
O spread β-ajustado pode ser modelado como um processo auto-regressivo de primeira
ordem (AR(1)): a variação do spread em cada instante é proporcional ao seu nível
no instante anterior. O coeficiente de proporcionalidade λ indica a velocidade de
reversão. Se λ é negativo, o spread reverte à média; se λ é próximo de zero, a
reversão é muito lenta; se λ é positivo, o spread diverge.

A partir de λ, calcula-se o half-life: o número de barras necessárias para que o
spread percorra metade do caminho de volta à sua média após um desvio. A fórmula é:

half-life = −ln(2) / λ

Esse valor fornece uma referência natural para o tamanho da janela de entrada: se o
spread demora N barras para reverter metade do desvio, faz sentido que o modelo
observe pelo menos N barras de histórico para identificar essa dinâmica.

RESULTADO PARA PETR3/PETR4 E LIMITAÇÃO ENCONTRADA
───────────────────────────────────────────────────
Para PETR3/PETR4, λ = −0,000015, resultando em half-life de 45.058 barras. Esse
valor é estatisticamente válido — a reversão ocorre, mas é extremamente lenta —
porém é impraticável como janela de entrada: com 45.058 barras, o conjunto de
treinamento teria pouquíssimas sequências e os modelos não aprenderiam nada.

Esse resultado é, aliás, coerente com os 28 defasagens selecionadas pelo ADF: o
spread tem autocorrelação persistente e reverte lentamente, o que é típico de
dados intraday de alta frequência onde o ruído de curto prazo domina.

SOLUÇÃO IMPLEMENTADA (FALLBACK)
────────────────────────────────
Quando o half-life calculado excede o limite seguro para o tamanho do dataset, o
sistema utiliza automaticamente o número de defasagens selecionadas pelo teste ADF
como referência alternativa. O ADF seleciona o número de defasagens pelo critério
de informação AIC, que busca o modelo mais parcimonioso que ainda captura a estrutura
de autocorrelação do spread. Esse valor é portanto estatisticamente derivado dos
dados, ainda que de forma indireta.

Para PETR3/PETR4, o sistema utilizou 28 barras (lags do ADF) em vez de 45.058
(half-life), com uma mensagem explícita no console informando o fallback e o motivo.

EFEITO NO SISTEMA
─────────────────
Com INPUT_CHUNK_LENGTH = 28 em vez de 50 (padrão anterior), o conjunto de teste
disponível aumentou significativamente, pois menos barras são "consumidas" pela
janela de entrada. Isso resultou em quase o dobro de observações de teste, tornando
as métricas de desempenho das estratégias mais representativas. O lucro total da
estratégia ground truth (que representa o teto teórico de desempenho) passou de
131 para 264, refletindo o maior número de barras avaliadas.

CONEXÃO COM O FEEDBACK DO PROFESSOR
─────────────────────────────────────
O professor solicitou um estudo de sensibilidade a hiperparâmetros, testando
diferentes valores de janela de entrada e medindo o impacto no desempenho. A
incorporação do half-life responde a essa solicitação fornecendo uma justificativa
estatística para os valores testados: em vez de variar a janela arbitrariamente
(por exemplo, 25, 50, 100), varia-se em torno do valor estatisticamente motivado
(half-life ou lags do ADF), o que dá significado interpretável às variações.


================================================================================
ALTERAÇÃO 4 — THRESHOLDS DERIVADOS DO DESVIO-PADRÃO DO RATIO
================================================================================

PROBLEMA QUE RESOLVE
─────────────────────
As estratégias supervisionadas comparam o sinal gerado pelo modelo contra um limiar
de decisão (threshold): apenas se o sinal ultrapassar o limiar é que uma operação é
executada. Os thresholds anteriores eram valores fixos, iguais para todos os pares e
datasets, escolhidos sem relação com a magnitude das variações de preço observadas.

Um threshold de 0,0005 pode ser restritivo demais para um par de ações com alta
volatilidade (resultando em poucas operações) e permissivo demais para um par de
baixa volatilidade (resultando em operações em qualquer variação mínima de preço).
Em ambos os casos, os thresholds fixos distorcem a comparação entre pares distintos
e entre modelos treinados em períodos diferentes.

COMO FUNCIONA
─────────────
O sistema calcula a variação do ratio (preço_A / preço_B) entre cada instante e o
instante anterior, ao longo de todo o conjunto de treinamento. O desvio-padrão σ
dessas variações representa a volatilidade típica do ratio naquele par e período.

Os quatro thresholds são definidos como múltiplos de σ:
[0, 0,5σ, 1,0σ, 2,0σ]

Essa escala tem interpretação estatística direta: threshold de 1σ significa que o
modelo só entra na operação quando o sinal previsto é maior que um desvio-padrão
das variações típicas do ratio, o que corresponde a uma variação estatisticamente
relevante e não apenas ruído aleatório.

IMPORTANTE — SEM VAZAMENTO DE DADOS
─────────────────────────────────────
O desvio-padrão σ é calculado exclusivamente sobre o conjunto de treinamento, nunca
sobre os dados de teste. Isso garante que os thresholds não se beneficiam de
informações do futuro, mantendo a integridade metodológica do experimento.

RESULTADO PARA PETR3/PETR4
───────────────────────────
σ = 0,000693, resultando nos thresholds [0,0; 0,000347; 0,000693; 0,001387].
Os thresholds anteriores eram [0; 0,00025; 0,0005; 0,001]. A diferença é pequena
para PETR3/PETR4, mas os novos valores são específicos do par e do período, enquanto
os anteriores eram genéricos e arbitrários.

EFEITO NO SISTEMA
─────────────────
Todos os experimentos futuros com diferentes pares (ITAU3/ITAU4, BBDC3/BBDC4)
terão thresholds automaticamente ajustados à volatilidade daquele par específico,
tornando os resultados comparáveis entre si de forma mais justa. A comparação entre
pares deixa de ser distorcida por thresholds calibrados para apenas um deles.


================================================================================
RELAÇÃO ENTRE AS ALTERAÇÕES E O RESULTADO OBSERVADO
================================================================================

As quatro alterações formam um pipeline coerente de configuração baseada em dados:

1. Os três testes estatísticos (Pearson, Engle-Granger, ADF) validam o par e
   extraem dois parâmetros chave: o hedge ratio β e o número de defasagens do ADF.

2. O hedge ratio β é aplicado ao cálculo do spread, tornando-o estatisticamente
   correto em vez de depender da suposição arbitrária β = 1.

3. O half-life (ou as defasagens do ADF como fallback) define a janela de entrada
   dos modelos e a janela de normalização do spread, alinhando a escala temporal
   do modelo à dinâmica real do par.

4. O desvio-padrão do ratio no treinamento define os thresholds de decisão,
   tornando-os proporcionais à volatilidade observada de cada par.

O resultado é um sistema que se auto-configura a partir das características
estatísticas do par analisado, sem necessidade de ajuste manual de hiperparâmetros
para cada dataset. Isso é relevante academicamente porque torna o método
generalizável: aplicar o sistema a um novo par (como ITAU3/ITAU4) produz
automaticamente uma configuração estatisticamente fundamentada.


================================================================================
INTERPRETAÇÃO DOS RESULTADOS DAS ESTRATÉGIAS
================================================================================

HIERARQUIA ESPERADA (CONFIRMADA)
──────────────────────────────────
Ground Truth >> Híbrida > Reversão à Média > Previsão Pura > Momentum ≈ 0

Ground Truth representa o desempenho máximo teórico, assumindo previsão perfeita
do futuro. É o teto de referência, não uma estratégia aplicável na prática.

A estratégia Híbrida exige que tanto o sinal do modelo quanto o sinal de preço
apontem na mesma direção antes de executar uma operação. Isso reduz o número de
operações mas aumenta a qualidade de cada uma, resultando no maior lucro por
operação e no maior Sharpe ratio entre as estratégias de ML.

A estratégia de Reversão à Média combina o sinal de reversão do preço com a
previsão do modelo. Gera mais operações que a Híbrida e é mais robusta à qualidade
do modelo porque parte de um sinal de preço já informativo.

A estratégia de Previsão Pura depende inteiramente da qualidade do modelo. Com
poucos epochs de treinamento, produz sinais próximos do aleatório.

Momentum próximo de zero é o resultado esperado e correto para um par cointegrado.
Momentum é a estratégia certa para séries com tendência; para séries estacionárias
(como o spread de um par cointegrado), momentum sistematicamente apostaria na
direção errada e deveria produzir lucro nulo ou negativo.

SOBRE OS RESULTADOS COM N_EPOCHS = 3
──────────────────────────────────────
Os resultados apresentados foram obtidos com apenas 3 epochs de treinamento, para
validação rápida do pipeline. A matriz de confusão mostra taxa de verdadeiros
positivos abaixo de 1%, o que indica que o modelo não aprendeu padrões preditivos
com esse nível de treinamento. Isso é esperado e não representa um problema das
alterações implementadas — é simplesmente insuficiência de treinamento.

Para os resultados finais do artigo, o treinamento deve ser realizado com 50 epochs
ou mais. As métricas estruturais (hierarquia das estratégias, lucro por operação
da estratégia Híbrida, Sharpe ratio do Ground Truth) já se manifestam corretamente
com 3 epochs, confirmando que o pipeline está funcionando como esperado.

O fato de a estratégia Híbrida manter Sharpe ratio de 0,14 mesmo com modelo mal
treinado é uma observação relevante para o artigo: a Híbrida é robusta à qualidade
do modelo porque depende da concordância entre dois sinais independentes, e o sinal
de preço por si só já carrega informação suficiente para gerar operações positivas
quando confirmado pelo modelo.

  Implementado: os três testes (Pearson, Engle-Granger, ADF) rodam automaticamente

  justifica ou rejeita o par com base em evidência estatística.

ITEM 2 — Estudo de sensibilidade a hiperparâmetros
  Parcialmente implementado: o half-life e os lags do ADF fornecem uma referência
  estatística para o INPUT_CHUNK_LENGTH, e o σ do ratio fornece referência para os

  esses parâmetros em torno dos valores de referência e medir o impacto no desempenho,

  automaticamente para ITAU3/ITAU4 e BBDC3/BBDC4 com o mesmo comando.
